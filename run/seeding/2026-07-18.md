## ch2 Rodrigo Recio (PT-BR)

### 1. TabNews — post
**Título:** Você paga R$300/mês de API e podia rodar local — a conta que quase ninguém faz
**Corpo:**
R$300/mês de API de LLM dá ~R$3.600/ano. Uma 3060 12GB usada (~R$1.000) roda um 7B/13B
quantizado (Q4_K_M) a 40+ tokens/s pra extração, classificação e rascunho — tarefas que somam a
maior parte do custo de API de muita gente. Não é "local substitui tudo": chamadas de raciocínio
pesado continuam melhores na nuvem. É sobre mover o volume barato pra máquina e pagar API só onde
ela ganha. A conta economia-vs-cilada em 40s: https://youtu.be/EXskv-1fVVU
(é do meu canal — série diária sobre rodar IA em hardware de gente normal)

### 2. r/brdev — comment em thread fresca sobre custo de API / rodar local
A pergunta certa não é "local é mais barato?" e sim "quais das minhas chamadas são baratas?".
Extração/classificação/rascunho em 7B quantizado local cobre o grosso do volume; raciocínio
longo fica na API. Quem move só o volume barato pra uma GPU usada costuma cortar 60-80% da fatura
sem perder qualidade onde importa. Fiz um short com a conta: https://youtu.be/EXskv-1fVVU

### 3. LinkedIn (Rodrigo) — post
"Comprei uma RTX cara e mesmo assim trava." Na maioria dos casos o gargalo não é a GPU — é VRAM
insuficiente pro modelo (swap pra RAM), CPU/IO no pré-processamento, ou um contexto grande demais
estourando a memória. Trocar de placa sem medir onde está o gargalo é queimar dinheiro. Mostrei
como achar o gargalo real antes de comprar: https://youtu.be/8VGJ7jdikOw
