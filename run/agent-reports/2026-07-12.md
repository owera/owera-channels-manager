## Learn
- **The re-consent wasn't enough — the app was un-granting itself.** Any scope added
  at consent time was being stripped at the first token refresh. This also explains
  why analytics kept working (that path pins the full CONSENT_SCOPES): the two
  paths requested different scopes off the same refresh token. Lesson recorded here
  because it will recur with any future scope addition: **the grant lives in the
  refresh token; what you request at refresh time is what you get.**
