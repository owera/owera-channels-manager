## ch2 Rodrigo Recio (PT-BR)

### 1. TabNews — post
**Título:** Você comprou 32GB de RAM pra rodar IA local e continua lento — o gargalo quase nunca é a RAM

**Corpo:**
Vejo muita gente montando PC pra rodar modelo local, enchendo de RAM (32, 64GB) e depois
reclamando que o Ollama arrasta. A RAM ajuda a *carregar* o modelo, mas quem processa o token
é a GPU — e o que importa de verdade é a **VRAM**. Se o modelo não cabe na VRAM, ele "transborda"
pra RAM/CPU e a velocidade despenca (às vezes 10x mais lento), independente de quanta RAM você tem.

Regra prática pra não gastar errado: primeiro veja o tamanho do modelo quantizado (um 7B em
Q4 pede ~5-6GB de VRAM, um 13B ~9-10GB). Compre VRAM pra caber o modelo que você QUER rodar,
não RAM. Uma 3060 de 12GB roda muito mais liso que um monstro de 64GB de RAM com GPU fraca.

Fiz um vídeo curto explicando esse gargalo (com os números de quanto cada faixa de modelo
precisa): https://youtube.com/watch?v=xpCy4qk7f3Q — queria saber onde vocês esbarraram nesse
teto de VRAM.

### 2. r/brdev — comment (encontre uma thread fresca sobre "montar PC pra IA" / GPU / rodar LLM local)
> Se ninguém falar de hardware, comente numa thread de "vale a pena rodar modelo local?".

O erro mais caro que vi (e cometi) montando setup pra IA local: otimizar RAM em vez de VRAM.
RAM carrega, VRAM processa. Modelo que não cabe na VRAM transborda pra CPU e fica lento demais
pra valer a pena. Se o orçamento é curto, GPU usada com mais VRAM (3060 12GB, por ex.) rende
muito mais que placa nova cara com pouca memória — só cheque garantia e histórico de mineração.

Fiz um vídeo comparando o custo real de GPU usada pra IA (economia x cilada):
https://youtube.com/watch?v=uhTU9Midi6Q — mas o resumo tá aí se quiser só o TL;DR.

## ch1 Owera Software (EN)

### 1. r/LocalLLaMA or r/mcp — comment (find a fresh thread on MCP / connecting tools to an agent)
The thing that made MCP finally click for me: stop thinking of it as "another framework"
and think **USB-C for AI**. Before, every tool you wanted your agent to use was a bespoke
integration you wrote, tested, and maintained. MCP is the one port — you expose a tool once
via an MCP server and any MCP-aware client (Claude, your IDE, your agent) can plug in without
custom glue. The win isn't magic capability, it's that N tools × M clients stops being N×M
integrations and becomes N+M.

Made a 60-sec explainer that lands the USB-C analogy with a concrete server example:
https://youtube.com/watch?v=w2jZ_D8y-R8 — curious whether the analogy holds up for people
already running MCP servers in anger.

### 2. dev.to or r/LocalLLaMA — short post (Copilot/Cursor pricing wedge — the ch1 converter)
**Title:** "Cheap" AI coding subscriptions feel free until the credit meter flips — here's the math

**Body:**
The $20/mo AI coding tools advertise a flat price, but under the hood many now meter
"premium requests" / credits — and heavy users hit the wall mid-month, then it's overage or
throttling. The honest comparison isn't sticker price, it's **cost per unit of actual work**:
how many real requests you get before the meter bites, and what a request costs after.

I broke down where the credit model actually costs you (and when a model-agnostic / API-metered
route is genuinely cheaper): https://youtube.com/watch?v=IfqX0JRKYcE — would like to hear what
people are paying at heavy daily use, because the marketing numbers don't match my bills.
