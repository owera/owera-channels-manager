**Targeted: R3 code-beat indentation → shipped `13be882`.** Root cause traced two layers deep:
1. `storyboard._TYPE_DOCS["code"]` never told the LLM to encode indentation as literal leading
   spaces (fixed, but insufficient alone).
2. **The real bug:** `_chars_clip()` — the shared field-clamp helper used for every beat field
   (cue, emoji, code lines, command output, etc.) — unconditionally calls `.strip()`, which
   destroys leading whitespace even if the LLM *did* emit indented code. Confirmed by re-rendering
   after step 1 alone: `return` was still flush-left with the LLM re-generating a fresh (still
   correct-in-intent, still stripped) snippet.
   Fix: added `_code_line_clip()` (rstrip-only — trailing whitespace trimmed, leading preserved)
   and pointed `code.lines` at it specifically; every other `_chars_clip()` call site (cue, titles,
   node ids, etc., where leading/trailing whitespace is noise, not signal) is untouched.
