### 1. TabNews — post
**Título:** Comprou Mac M3 "pra rodar IA local" — o que ninguém te conta antes da compra
**Corpo:**
Vejo muita gente gastando R$15-20k num Mac M3 achando que memória unificada resolve tudo em
IA local. A real: pra inferência de modelos 7B-13B quantizados, funciona muito bem — mas o

## ch1 Owera Software (EN)

### 1. r/LocalLLaMA — comment (find a fresh RAG-quality / retrieval thread)
**Draft:**
Most "my RAG returns garbage" threads end up being chunk-size problems, not model problems.
Fixed-size chunking cuts sentences mid-thought, embeddings index fragments, retrieval returns
noise. Splitting on semantic structure (paragraphs/sections) fixed more for us than any
model swap. I made a 40s visual explainer of the failure mode if useful:
https://youtu.be/jpjX86HnhlY (my channel, fwiw). The TL;DR works without the click though:
chunk by meaning, not by count.

### 2. dev.to — short post
**Title:** Stop trusting LLM retries — your agent needs circuit breakers
**Body:**
Every agent framework defaults to "retry on failure". But when a provider degrades, retries
amplify the outage: N agents × M retries = you DDoS yourself and burn tokens on calls that
were never going to succeed. Classic distributed-systems answer: circuit breakers. Trip after
K consecutive failures, back off, half-open probe, close on success. Works for LLM calls
exactly like it works for microservices.

60-second visual version: https://youtu.be/5mftL2wSHAc
Longer piece coming on the channel this week — comparing breaker placement (per-tool vs
per-provider) in LangChain vs CrewAI stacks.
