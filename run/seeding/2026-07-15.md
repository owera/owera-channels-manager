## ch2 Rodrigo Recio (PT-BR)

### 1. TabNews — post
**Título:** GPU cara e inferência lenta: o gargalo quase nunca é onde você acha
**Corpo:**
Comprar uma GPU melhor e ver o Ollama continuar lento é mais comum do que parece — e o
motivo quase nunca é a GPU. Os três suspeitos reais, na ordem em que valem ser checados:

1. **O modelo não coube na VRAM e ninguém avisou.** O Ollama faz offload silencioso de
   camadas pra RAM; a GPU fica a 30% e a CPU vira o gargalo. `ollama ps` mostra o split —
   se aparecer qualquer % em CPU, é isso.
2. **Contexto acumulado.** Cada turno da conversa reprocessa o histórico inteiro; com
   `num_ctx` alto o prefill engole segundos antes do primeiro token.
3. **Quantização errada pro caso.** Q8 num modelo que caberia em Q4_K_M dobra o tráfego
   de memória sem ganho perceptível de qualidade em inferência local.

Gravei um diagnóstico em vídeo (curto, direto) mostrando como identificar cada caso:
https://youtu.be/-r4helTl_J4

Qual desses três já te pegou? Aposto no offload silencioso — é o mais traiçoeiro.

## ch1 Owera Software (EN)

### 1. dev.to — post
**Title:** Stop benchmarking agent frameworks on the demo — benchmark the failure path
**Body:**
Every LangChain-vs-CrewAI thread compares time-to-first-agent. But the decision that
matters is what happens on step 7 of a 12-step run when a tool call times out: which
framework lets you checkpoint, retry *that step* with a different policy, and stream
partial progress to the user while it recovers?

My short checklist before committing to either: (1) can I resume a crashed run without
re-executing paid LLM calls? (2) can retry policy differ per tool? (3) does streaming
survive custom control flow? (4) can I eject one component without rewriting the graph?

I walked through both frameworks against exactly this checklist in a longer video:
https://youtu.be/FVS_ADw98vo — curious which failure-path features people actually use
in production, and which are résumé-driven.

### 2. LinkedIn (Owera) — post
"Which agent framework?" is the wrong first question. "What happens when step 7 fails?"
is the right one. LangChain and CrewAI answer it very differently — checkpointing, per-tool
retries, and streaming under custom control flow are where they split. Full comparison:
https://youtu.be/FVS_ADw98vo
