Losers lead with the solution or feature ("Mastering…", "Optimize…"). Predicted: switching to viewer-first hooks should improve views over next 7 days.

### Code change — Title hook prompt strengthened (`app/services/video_gen.py`)

**What changed:** The idea generation prompts for both short and long formats now explicitly:
1. Require leading with the viewer's situation/mistake before the solution ("Your AI Hallucinates Because…" not "Mastering AI Outputs…")
2. Add specific winning patterns: root-cause curiosity, behavior interrupt, relatable setup, demystification
3. Prohibit opener words that correlate with low clicks: `Mastering`, `Deep Dive`, `Optimize`, `Complete Guide`, `Cut X%`

**Why:** The #1 and #2 videos on each channel all open with the viewer's problem or situation. The 0-view videos all open with the solution, method, or feature. This is the clearest signal in the data.

**Verified by exercising `generate_ideas()` directly:**
```
uv run python -c "from app.services.video_gen import generate_ideas; ..."
Output:
 - Why Your Prompts Work Once Then Break Forever
 - Stop Writing Longer Prompts — Add Examples Instead
 - Your AI Hallucinates Because You Skipped This One Step
 - Context Windows Aren't Magic — They're Just Expensive Memory
 - Your AI Ignores Half Your Instructions — Here's Why
```
All 5 generated titles use viewer-first hooks — none use the prohibited openers. Change is live and producing the intended pattern.
